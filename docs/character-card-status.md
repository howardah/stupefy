# Character Card Status

This document summarizes the character deck as implemented in the migrated Nuxt/Vue gameplay code.

Status meanings:

- `Yes`: a character-specific gameplay rule is clearly implemented.
- `Partial`: there is some evidence of support, but the repo does not fully describe or fully enforce the power.
- `No`: no character-specific gameplay rule was found in the migrated code.

Important limitation:

- Most character powers are now implemented in the migrated codebase.
- The remaining `Partial` entries reflect rules where the original tabletop text still requires a concrete implementation assumption in the current code.

Primary code references:

- `utils/stupefy-decks.ts`
- `utils/gameplay/card-rules.ts`
- `utils/gameplay/events.ts`
- `utils/gameplay/turn-cycle.ts`
- `utils/gameplay/board-ui.ts`
- `docs/character-power-audit.md`

## Notes

- `Albus Dumbledore` is marked `Partial` because the current migration suppresses the main opposing tableau effects that matter in range/defense resolution, but it is not yet a blanket engine-wide "ignore every opposing tableau effect" rule.
- `Minerva McGonagall` is marked `Partial` because the implementation currently treats "gray cards" as the non-spell utility/tableau subset; the original tabletop definition is not encoded more precisely in the recovered code.
- `Voldemort` remains `Partial`: the horcrux-style health gain is implemented and the standard cloak/cabinet hide options are blocked, but the "except that of the deathly hallows" wording still needs a more explicit rules-level implementation.
- `Mad-Eye Moody` still has no migrated gameplay/UI rule that reveals Death Eater roles to the Moody player.
